## 개요

기존에 대시보드를 만들려면 루커 스튜디오(Looker Studio)나 파워 BI(Power BI) 같은 대시보드 툴을 별도로 학습해야 했습니다. 하지만 AI 바이브코딩 시대에는 클로드코드를 활용하면 코딩을 전혀 모르는 비개발자도 구글 시트 데이터 기반의 커스텀 대시보드를 직접 만들 수 있습니다.

이 가이드에서는 다음 도구를 활용합니다:

- **클로드코드(Claude Code)**: AI 코딩 에이전트로, 자연어 지시만으로 코드를 작성
- **GSD 프레임워크(Get Shit Done)**: 클로드코드의 작업을 탐색 → 계획 → 실행 → 검증 단계로 체계화
- **대시보드 스타터킷**: 구글 시트 연동, UI 프레임워크, 프로젝트 가이드가 포함된 기본 템플릿
- **Vercel**: GitHub 연동으로 간편하게 배포할 수 있는 호스팅 서비스

## 전체 워크플로우

대시보드 제작은 크게 세 단계로 진행됩니다:

**1단계 — 개발 환경 셋팅**: VS Code에 클로드코드를 설치하고, MCP 서버와 Skills 같은 보조 도구를 셋팅합니다. 작업장을 세팅하고 도구를 배치하는 단계입니다.

**2단계 — 클로드코드로 코드 제작**: GSD 프레임워크의 탐색 → 계획 → 실행 → 커밋/저장 워크플로우에 따라 대시보드를 단계별로 제작합니다. 스타터킷 위에서 작업하여 기초 작업의 반복을 줄입니다.

**3단계 — Vercel 배포**: 완성된 대시보드를 GitHub와 연동하여 Vercel로 배포합니다. 환경변수 설정만 해주면 외부에서도 접속 가능한 대시보드가 됩니다.

## 사전 준비사항

- VS Code (코드 편집기)
- Node.js (클로드코드 실행 환경)
- Git (코드 버전 관리)
- 구글 계정 (GCP 서비스 어카운트 및 시트 API용)
- Claude 계정 (클로드코드 로그인용)
- GitHub 계정 (배포용)
- Vercel 계정 (배포용)

## 개발 환경 셋팅

### 1. VS Code 설치

[VS Code 공식 웹사이트](https://code.visualstudio.com/)에서 운영체제에 맞는 버전을 다운로드하여 설치합니다.

VS Code는 코드 편집기로, 클로드코드가 작성한 코드를 확인하고 파일 구조를 보거나 설정 파일을 수정할 때 사용합니다. 비개발자도 코드를 직접 작성할 필요는 없지만, 프로젝트 관리를 위해 설치를 권장합니다.

### 2. Node.js 설치

[Node.js 공식 웹사이트](https://nodejs.org/)에서 운영체제에 맞는 버전을 설치합니다.

설치 확인:
```bash
node -v
```
버전 번호가 출력되면 정상 설치된 것입니다.

### 3. Git 설치

Git은 코드 버전 관리 도구입니다. GSD 프레임워크를 사용하면 각 단계가 끝날 때마다 작업 내용을 커밋(저장)하게 되는데, 문제가 생기면 이전 세이브 포인트로 돌아갈 수 있어 매우 유용합니다.

**맥:**
```bash
brew install git
```

**윈도우:** [Git 공식 사이트](https://git-scm.com/)에서 다운로드하여 설치

## 클로드코드 설치

VS Code를 열고 터미널을 실행한 후 다음 명령어로 클로드코드를 설치합니다.

**맥:**
```bash
curl -fsSL https://claude.ai/install.sh | bash
```

**윈도우:**
```powershell
irm https://claude.ai/install.ps1 | iex
```

설치 완료 후 터미널에서 `claude`를 입력하면 클로드코드가 실행됩니다. 처음에 계정 로그인을 한 번 진행하면 바로 사용 가능합니다.

종료: `/exit`

## MCP 서버 설치

클로드코드가 더 똑똑하게 작업할 수 있도록 MCP 서버 두 개를 설치합니다.

### Context7 설치

Context7은 클로드코드가 개발 시 필요한 공식 문서를 자동으로 찾아서 참고할 수 있게 해주는 MCP 서버입니다. 개발자가 코딩할 때 공식 문서를 참고하는 것처럼, 클로드코드에게도 레퍼런스를 제공해줍니다.

```bash
claude mcp add --scope user --header "CONTEXT7_API_KEY: YOUR_API_KEY" --transport http context7 https://mcp.context7.com/mcp
```

> `YOUR_API_KEY` 부분에 Context7에서 발급받은 API 키를 입력합니다. (optional)

### Playwright 설치

Playwright는 클로드코드가 실제 웹 브라우저 화면을 직접 보고 확인할 수 있게 해주는 MCP 서버입니다. 대시보드 화면이 의도한 대로 나왔는지 확인하고, 버그를 찾아서 수정까지 할 수 있습니다.

```bash
claude mcp add playwright npx @playwright/mcp@latest
```

## 스타터킷 설치 및 설정

매번 처음부터 프로젝트 구조를 잡고 구글 시트 연동 설정을 반복하는 것은 비효율적입니다. 스타터킷에는 다음이 포함되어 있습니다:

- 구글 시트 연동을 위한 기본 설정
- 대시보드 UI 프레임워크
- 클로드코드가 참고할 수 있는 프로젝트 가이드 문서

### 1. 스타터킷 클론

상위 폴더로 이동하여, 아래 커맨드를 실행합니다.

```bash
git clone https://github.com/citizendev9c/googlesheet-dashboard-starterkit.git my-project-name
cd my-project-name
npm install
```

### 2. Git 초기화

원본 스타터킷과의 연결을 끊고 새 프로젝트로 시작합니다:

```bash
rm -rf .git
git init
git add .
git commit -m "init: 스타터킷에서 시작"
```

### 3. 환경변수 설정

```bash
cp .env.example .env.local
```

`.env.local` 파일에 구글 시트 연동에 필요한 정보를 입력합니다. GCP 콘솔에서 서비스 어카운트를 생성하고 시트 API를 활성화한 후 키를 발급받아 환경변수에 넣어줍니다.

> GCP 설정 방법을 모르겠으면 클로드코드에게 "GCP에서 구글 시트 API 서비스 어카운트 어떻게 설정해?"라고 질문하면 단계별로 안내해줍니다.

## GSD 프레임워크 설치

GSD(Get Shit Done) 프레임워크는 클로드코드의 작업을 체계적으로 진행할 수 있게 도와줍니다. 리서치/탐색 → 계획 수립 → 코드 제작 → 저장/검증 워크플로우를 손쉽게 구현해줍니다.

클로드코드를 `/exit`으로 종료한 후 터미널에서 설치합니다:

```bash
npx get-shit-done-cc@latest
```

설치 후 클로드코드를 실행하면 `gsd:` 로 시작하는 커맨드를 사용할 수 있습니다.

### 주요 GSD 커맨드

| 커맨드 | 역할 |
|--------|------|
| `gsd:map-codebase` | 프로젝트 전체 코드 구조 분석 |
| `/gsd:new-project` | 새 프로젝트 생성 및 초기 설정 |
| `/gsd:discuss-phase N` | Phase N 탐색 및 방향 논의 |
| `/gsd:plan-phase N` | Phase N 구체적 작업 계획 수립 |
| `/gsd:execute-phase N` | Phase N 코드 작성 실행 |
| `/gsd:verify-work N` | Phase N 결과물 검증 |
| `/gsd:audit-milestone` | 전체 마일스톤 감사 |
| `/gsd:complete-milestone` | 마일스톤 완료 및 버전 저장 |

## 시트 데이터 구조

대시보드로 만들 구글 시트는 세 개의 시트로 구성됩니다:

**거래내역 시트**: 주문일, 상품명, 수량, 금액, 고객 정보 등 거래 데이터

**상품 시트**: 상품별 카테고리, 단가, 마진율 등 상품 마스터 데이터

**고객 시트**: 고객별 등급, 가입일, 지역 등 고객 마스터 데이터

> 실제 사업하시는 분들도 비슷한 형태로 시트를 관리하고 계실 것입니다. 이 데이터를 기반으로 매출 현황, 상품별 분석, 고객별 분석을 한눈에 볼 수 있는 대시보드를 만듭니다.

## 대시보드 제작 (Phase별 진행)

클로드코드를 실행하고 먼저 스타터킷 구조를 분석합니다:

```
gsd:map-codebase
```

그 다음 새 프로젝트를 생성합니다:

```
/gsd:new-project
```

프로젝트 생성 시 다음과 같이 요구사항을 전달합니다:

```
구글 시트에 구씨컴퍼니라는 회사 데이터가 있습니다.
이 시트 데이터를 기반으로 매출 대시보드를 제작하고 싶습니다.
시트가 세 개로 구성이 되어 있습니다.
첫 번째는 거래내역 시트입니다. 주문일, 상품명, 수량, 금액, 고객 정보 이런 것들이 들어가 있고요.
두 번째는 상품 시트입니다. 상품별 카테고리, 단가, 마진율 같은 정보가 있습니다.
세 번째는 고객 시트입니다. 고객별 등급, 가입일, 지역 이런 정보들이 담겨 있습니다.
이 데이터를 기반으로 매출 현황, 상품별 분석, 고객별 분석을 한눈에 볼 수 있는 대시보드를 만들어줘.
```

GSD 프레임워크가 기본 정보 질문, 모드 설정, 추가 리서치를 진행한 후 다음 문서들을 자동 생성합니다:

- `ARCHITECTURE.md` — 아키텍처 설계
- `FEATURES.md` — 기능 명세
- `PITFALLS.md` — 주의사항
- `STACK.md` — 기술 스택
- `SUMMARY.md` — 프로젝트 요약
- `ROADMAP.md` — 로드맵
- `REQUIREMENTS.md` — 요구사항 정의

### Phase 1: 데이터 기반 + 인증

구글 시트에서 데이터를 가져오는 파이프라인과 로그인 인증 기능을 구현하는 단계입니다. 눈에 보이는 화면은 아직 없지만, 대시보드의 기반이 되는 핵심 인프라를 구축합니다.

**탐색:**
```
/gsd:discuss-phase 1
```

요청 예시:
```
구글 시트에 거래내역, 상품, 고객, AllowedEmails 네 개 시트가 있는데,
이 데이터를 batchGet으로 한 번에 가져오는 데이터 레이어를 만들어줘.
그리고 AllowedEmails 시트에 등록된 이메일만 구글 로그인으로 접근할 수 있게 인증도 구현해줘.
```

> **팁**: AllowedEmails 시트를 별도로 만들어 접근 가능한 이메일 리스트를 관리하면, 코드 수정 없이 시트에서 이메일만 추가/삭제하여 접근 권한을 관리할 수 있습니다.

**계획:**
```
/gsd:plan-phase 1
```

클로드코드가 작성하는 계획 내용: 시트 4개를 batchGet으로 한 번에 가져오는 API 구성, 60초 캐싱 설정, 시트 페칭 실패 시 폴백 처리, 구글 OAuth 인증 구현 등

**실행:**
```
/gsd:execute-phase 1
```

시트 데이터를 가져와서 타입 안전하게 파싱하는 로직, 캐싱 레이어, 구글 로그인 화면까지 자동으로 생성됩니다.

**검증:**
```
/gsd:verify-work 1
```

시트 데이터가 잘 불러와지는지, 등록된 이메일로만 로그인되는지, 미등록 이메일은 제대로 차단되는지를 확인하고 문제가 있으면 수정합니다.

### Phase 2: 매출 현황 대시보드

드디어 눈에 보이는 메인 대시보드 화면을 만드는 단계입니다.

**탐색:**
```
/gsd:discuss-phase 2
```

요청 예시:
```
메인 대시보드 상단에 총매출, 주문 수, 평균 주문액 세 가지 핵심 KPI를 카드 형태로 보여주고,
전월 대비 변화율도 색상으로 구분해서 표시해줘.
그 아래에는 월별 매출 바 차트를 넣고, 기간 필터로 월별, 분기별, 전체를 선택할 수 있게 해줘.
그리고 사이드바에 매출, 상품, 고객 세 개 메뉴를 만들어서 탭 이동이 가능하게 구성해줘.
```

**계획 → 실행:**
```
/gsd:plan-phase 2
/gsd:execute-phase 2
```

KPI 카드 컴포넌트, 차트 라이브러리 연결, 기간 필터에 따른 데이터 갱신 로직이 자동으로 구현됩니다. Playwright MCP를 통해 클로드코드가 직접 브라우저에서 화면을 확인하고 레이아웃 문제를 수정합니다.

**검증:**
```
/gsd:verify-work 2
```

결과물: KPI 카드 3개, 월별 매출 바 차트, 기간 필터, 사이드바 네비게이션이 포함된 메인 대시보드

---

### Phase 3: 상품 분석

어떤 상품이 잘 팔리고 있는지 분석하는 화면을 만드는 단계입니다.

**탐색:**
```
/gsd:discuss-phase 3
```

요청 예시:
```
상품 분석 탭에 카테고리별 매출 비중을 도넛 차트로 보여주고,
매출 Top N 상품을 수평 바 차트로 보여줘.
그리고 밑에는 상품별 매출 테이블을 넣어서 정렬이랑 검색이 가능하게 해주고,
CSV 다운로드 기능도 추가해줘. 한글 엑셀에서도 깨지지 않게 처리해줘.
```

> **팁**: CSV 다운로드를 넣으면 대시보드에서 데이터를 확인하다가 더 세부적으로 분석하고 싶을 때 엑셀로 내려받아서 작업할 수 있습니다.

**계획 → 실행 → 검증:**
```
/gsd:plan-phase 3
/gsd:execute-phase 3
/gsd:verify-work 3
```

결과물: 카테고리별 도넛 차트, 상위 상품 바 차트, 정렬·검색 가능한 상품 테이블, CSV 다운로드

---

### Phase 4: 고객 분석

고객이 누구인지, 어디에 분포해 있는지 파악할 수 있는 화면을 만드는 단계입니다.

**탐색:**
```
/gsd:discuss-phase 4
```

요청 예시:
```
고객 분석 탭에 고객 세그먼트별 매출 비중을 도넛 차트로 보여주고,
산업군별, 지역별 매출을 바 차트로 각각 보여줘.
매출 상위 고객 바 차트랑, 고객별 상세 매출 테이블도 넣어줘.
테이블은 상품 분석이랑 동일하게 정렬, 검색, CSV 다운로드가 가능하게 해줘.
```

> **팁**: 고객 분석을 세그먼트(등급), 산업군, 지역 세 가지 관점으로 나누면 "VIP 고객의 매출 기여도", "주문이 많은 산업", "주문이 많은 지역" 등 다양한 질문에 답할 수 있습니다.

**계획 → 실행 → 검증:**
```
/gsd:plan-phase 4
/gsd:execute-phase 4
/gsd:verify-work 4
```

결과물: 고객 세그먼트 도넛 차트, 산업군/지역별 바 차트, 상위 고객 바 차트, 고객별 상세 테이블

---

### 마일스톤 완료

Phase 1~4까지 모든 작업이 끝나면 마일스톤을 마무리합니다:

```
/gsd:audit-milestone
/gsd:complete-milestone
```

모든 작업이 하나의 버전(v1.0 MVP)으로 저장됩니다.

## Vercel 배포

완성된 대시보드를 외부에서도 접속할 수 있도록 Vercel로 배포합니다.

1. [Vercel](https://vercel.com/)에 로그인합니다.
2. GitHub 저장소를 Import합니다.
3. Deploy Options에서 환경변수(Environment Variables)를 `.env.local` 파일의 내용과 동일하게 설정합니다.
4. Deploy 버튼을 클릭하면 배포가 완료됩니다.

배포가 완료되면 생성된 URL로 어디서든 대시보드에 접속할 수 있습니다.

## 완성된 대시보드 기능

**메인 대시보드 (매출 현황)**
- 총매출, 주문 수, 평균 주문액 KPI 카드 (전월 대비 변화율 색상 표시)
- 월별 매출 추이 바 차트
- 기간 필터 (월별/분기별/전체)
- 사이드바 네비게이션

**상품 분석**
- 카테고리별 매출 비중 도넛 차트
- 매출 Top N 상품 수평 바 차트
- 상품별 매출 테이블 (정렬/검색/CSV 다운로드)

**고객 분석**
- 고객 세그먼트별 매출 비중 도넛 차트
- 산업군별·지역별 매출 바 차트
- 매출 상위 고객 바 차트
- 고객별 상세 매출 테이블 (정렬/검색/CSV 다운로드)

**기타**
- 구글 OAuth 인증 (AllowedEmails 시트 기반 접근 제어)
- 구글 시트 실시간 연동 (데이터 추가 시 대시보드 자동 반영)
- 60초 캐싱으로 API 호출 최적화

## 핵심 정리 및 팁

**클로드코드 활용**: 코딩을 몰라도 AI에게 자연어로 지시를 내려서 대시보드를 만들 수 있습니다.

**GSD 프레임워크**: 탐색 → 계획 → 실행 → 검증의 체계적인 단계로 진행하면, AI가 실수할 확률을 줄이고 원하는 결과물을 얻을 확률을 높일 수 있습니다.

**스타터킷 활용**: 재사용 가능한 템플릿을 미리 준비해두면, 비슷한 대시보드를 빠르게 만들어낼 수 있습니다.

**주의사항**: 클로드코드가 만능은 아닙니다. 복잡한 요구사항을 한꺼번에 던지면 결과물이 엉뚱하게 나올 수 있으므로, 작은 단위로 쪼개서 진행하고 각 단계마다 검증하는 습관이 중요합니다.
